# Cinesphere SEO Implementation Guide

## Overview

Comprehensive SEO implementation for cinesphere.app with dynamic metadata, structured data, and search engine optimization across all main content pages.

## Architecture

### 1. Configuration ([config/site.ts](config/site.ts))

**Single source of truth** for all site configuration constants:

- `SITE_URL`: Production domain (cinesphere.app)
- `SITE_NAME`: "Cinesphere"
- `SITE_DESCRIPTION`: Main site description
- `DEFAULT_OG_IMAGE`: Fallback image for pages without specific images
- `SOCIAL_LINKS`: Twitter, Instagram, GitHub profiles

**Usage:** Import and use in all metadata generators and throughout the app:

```typescript
import { SITE_URL, SITE_NAME } from "@/config/site";
```

### 2. SEO Utilities ([lib/seo-utils.ts](lib/seo-utils.ts))

**Centralized metadata generation** with consistent patterns across all pages:

#### Key Functions:

- **`generatePageMetadata()`** - Generic metadata generator for any page type
  - Handles title, description, canonical URLs
  - Generates OpenGraph images and Twitter Card metadata
  - Supports custom ogType (website, article, profile, etc.)
- **`generateMovieMetadata()`** - Movie-specific metadata
  - Includes release year in description
  - Poster as OG image
- **`generateReviewMetadata()`** - Review-specific metadata
  - Includes author and rating information
- **`generateProfileMetadata()`** - User profile metadata

  - Profile image as OG image
  - Bio as description

- **JSON-LD Schema Generators:**
  - `movieSchema()` - Schema.org Movie type with aggregate rating
  - `reviewSchema()` - Schema.org Review type with rating
  - `personSchema()` - Schema.org Person type for actors/crew

**Usage Pattern:**

```typescript
import { generatePageMetadata } from "@/lib/seo-utils";

return generatePageMetadata({
  title: "Page Title",
  description: "Page description",
  canonical: "https://cinesphere.app/page",
  ogImage: "https://image-url.jpg",
  ogType: "website", // or 'article', 'profile', etc.
  authors: ["Author Name"],
});
```

### 3. Root Layout ([app/layout.tsx](app/layout.tsx))

**Site-wide metadata and structured data:**

#### Exports:

- **`metadata`** - Global metadata including:

  - metadataBase (prevents duplicate content)
  - robots rules (index: true, follow: true)
  - OpenGraph defaults
  - Twitter Card defaults
  - Google-specific crawler rules
  - Icons and themes

- **`viewport`** - Device scaling and theme color

#### Structured Data:

- **Organization Schema** - Site identity, logo, social links
- **Website Schema** - Search action template for Google

**Key Code:**

```typescript
export const metadata: Metadata = {
  metadataBase: new URL(SITE_URL),
  robots: {
    index: true,
    follow: true,
    "max-image-preview": "large",
    "max-snippet": -1,
    "max-video-preview": -1,
    googleBot: {
      index: true,
      follow: true,
      "max-image-preview": "large",
      "max-snippet": -1,
      "max-video-preview": -1,
    },
  },
  openGraph: {
    /* ... */
  },
  twitter: {
    /* ... */
  },
};
```

### 4. Dynamic Sitemap ([app/sitemap.ts](app/sitemap.ts))

**Automatic sitemap generation** for search engine crawling:

- Homepage: priority 1.0 (most important)
- Main sections (feed, explore, upcoming): priority 0.9
- Content pages (movies, shows, crew): priority 0.8
- User profiles: priority 0.6
- Legal pages (privacy, terms): priority 0.3

**Generated at:** `https://cinesphere.app/sitemap.xml`

**Usage:** Automatically generated by Next.js - no manual action needed.

### 5. Robots Rules ([app/robots.ts](app/robots.ts))

**Environment-aware crawler control:**

#### Development/Staging:

- Blocks all crawlers with `User-agent: *; Disallow: /`

#### Production:

- Blocks sensitive paths: `/api/`, `/admin/`, `/_next/`, `/private/`, `/auth/`
- Allows crawlers to index all public content
- Points to sitemap for discovery

**Verified in:** Blocks production API routes and auth pages from being indexed.

## How It Works

### 1. When User Shares a Link (e.g., on Discord, Twitter, WhatsApp)

1. Social media crawlers hit the URL with their user-agent
2. Next.js serves the page with `generateMetadata()` output
3. Crawler extracts:
   - `<title>` tag (from `title`)
   - `<meta name="description">` (from `description`)
   - `<meta property="og:image">` (from `ogImage`)
   - `<meta property="og:type">` (from `ogType`)
   - `<meta property="og:url">` (canonical)
   - `<meta name="twitter:card">` (Twitter Card)
4. Social platform displays rich preview with image, title, and description

### 2. Search Engine Discovery

1. Googlebot visits `sitemap.xml` (linked in `robots.txt`)
2. Discovers all pages with priority levels
3. Crawls each page and indexes content
4. Uses structured data (Organization, Website, Movie, Review schemas)
5. Includes pages in search results with rich snippets

### 3. Dynamic Metadata Generation

Each page's `generateMetadata()` function:

1. Receives `params` (dynamic route segments like `[id]` or `[username]`)
2. Fetches required data from API actions
3. Determines appropriate images and descriptions
4. Returns Metadata object with all SEO fields

## Troubleshooting

**Issue: OG images not showing on social media**

- Verify image URL is publicly accessible
- Check TMDB image path is correct format: `https://image.tmdb.org/t/p/w1280/path`
- Use OpenGraph debugger: https://www.opengraph.xyz

**Issue: Metadata not updating after deployment**

- Clear social platform cache by sharing with `?utm_source=cache_bust` parameter
- Wait 24-48 hours for search engines to re-crawl
- Manually request re-index in Google Search Console

**Issue: Some pages showing generic description**

- Check if API call is failing (try-catch in generateMetadata)
- Verify action functions are exported correctly
- Check for typos in data field names

**Issue: Canonical URLs creating duplicates**

- Ensure each page's canonical URL is unique
- Verify no trailing slashes inconsistencies
- Use Google Search Console to test and remove duplicates
